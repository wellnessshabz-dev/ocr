# Tracking — Session Log & Checkpoint System

## What This Is

This directory tracks every agent session in the OCR project. When an agent
works on the codebase, it logs what it did, why, and where checkpoints are
so work can be resumed or rolled back.

## File Layout

| File | Purpose | Persistence |
|------|---------|-------------|
| `SESSION.md` | Active session tracker — goal, progress, decisions, context | Ephemeral (resets per session) |
| `LOG.md` | Chronological log of all sessions | Permanent (append-only) |
| `DECISIONS.md` | Decision log with rationale, alternatives, consequences | Permanent (append-only) |
| `CHECKPOINTS.md` | Registry mapping ckpt steps → git hashes | Regenerated by sync |
| `STATE.json` | Machine-readable mirror of SESSION.md | Regenerated by sync |
| `PROTOCOL.md` | Agent protocol — how to start/snap/sync/end sessions | Static |
| `tools/session.sh` | Shell script wrapping ckpt + tracking file updates | Static |

## How It Works

```
ckpt (git hidden branches)           ocr/tracking/ (session state)
         │                                      │
  ckpt start ───────────►  session.sh start ──► SESSION.md + STATE.json
  ckpt snap "msg" ──────►  session.sh snap ───► SESSION.md (progress)
         │                    + CHECKPOINTS.md (hash)
  ckpt steps ───────────►  session.sh sync ───► CHECKPOINTS.md + STATE.json
  ckpt end ─────────────►  session.sh end ────► LOG.md (append) + ckpt squash
```

### Quick Start for Agents

```
1. Read SESSION.md — understand current state
2. Run: ./ocr/tracking/tools/session.sh start "Goal: X" — begin tracking
3. Before each change: ./ocr/tracking/tools/session.sh snap "doing Y"
4. After each decision: write to DECISIONS.md (one entry)
5. At session end: ./ocr/tracking/tools/session.sh end
```
